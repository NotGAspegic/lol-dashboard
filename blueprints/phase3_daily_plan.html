Fix CORS + Verify Frontend ↔ API Connection
~1.5 hrsSetup
  • The 502 error from before is a CORS issue. Open backend/main.py and find your CORSMiddleware. Make sure allow_origins includes "http://localhost:3000" — this is your Next.js dev server.
  • Also add allow_credentials=True, allow_methods=["*"], allow_headers=["*"] to the CORS config. Without these, preflight OPTIONS requests from the browser will be rejected.
  • Restart the API container: docker compose restart api. Then test CORS directly from the browser console on localhost:3000: open DevTools → Console and run fetch("http://localhost:8000/api/v1/health").then(r => r.json()).then(console.log). Should return {status: "ok"}.
  • In frontend/.env.local, confirm: NEXT_PUBLIC_API_URL=http://localhost:8000. Then update frontend/src/lib/api.ts — make sure the axios base URL reads from this env var: baseURL: process.env.NEXT_PUBLIC_API_URL + "/api/v1".
  • Test the full roundtrip: update src/app/page.tsx to call searchSummoner("G2 Caps", "1323", "euw1") and render the result. Open localhost:3000 — you should see the JSON response on screen with no errors.
✦ Day 1 Deliverable

Summoner Search Page — Input + Debounce
~2 hrsUI
  • Create src/app/page.tsx as the home/search page. Replace the test code with a centered layout: app logo/name at top, a large search input below, and a region selector. This is the first thing users see.
  • Install debounce utility: npm install use-debounce. The search input should accept Name#TAG format. As the user types, debounce the value by 500ms before doing anything with it — this prevents firing on every keypress.
  • Create a RegionSelect component — a styled <select> with options: EUW, EUNE, NA, KR, BR, JP. Map each display name to its platform value (euw1, eun1, na1, kr, br1, jp1). Default to EUW since that's your region.
  • Parse the search input: split on # to get [gameName, tagLine]. Validate that both parts exist before enabling the search button. Show an inline hint: "Enter your Riot ID: Name#TAG" below the input.
  • On submit, call searchSummoner(). Handle two response cases: status === "onboarding" → show a loading state with a progress message, status === "ready" → redirect to /summoner/{puuid} immediately.
  • For the onboarding case, poll getTaskStatus(taskId) every 2 seconds using setInterval. Update the message based on the status: PENDING → "Fetching summoner data...", STARTED → "Ingesting match history...", SUCCESS → redirect to the profile page.
// src/app/page.tsx — search submit handler const handleSearch = async () => { const [gameName, tagLine] = query.split('#') if (!gameName || !tagLine) return setLoading(true) const result = await searchSummoner(gameName.trim(), tagLine.trim(), region) if (result.status === 'ready') { router.push(`/summoner/${result.puuid}`) return } // onboarding — poll until done const interval = setInterval(async () => { const status = await getTaskStatus(result.task_id!) setStatusMessage(status.message) if (status.status === 'SUCCESS') { clearInterval(interval) router.push(`/summoner/${result.puuid}`) } }, 2000) }

New FastAPI Endpoints — Stats Queries
~2 hrsCode
  • Before building the profile page UI, you need the data endpoints. Add GET /api/v1/summoners/{puuid}/matches to your FastAPI router. Accept query params: limit=20, offset=0. Return paginated match list with per-match stats for the given puuid.
  • The matches query joins match_participants + matches on gameId, filters by puuid, orders by gameStartTimestamp DESC. Return: gameId, championId, kills, deaths, assists, win, gameDuration, gameStartTimestamp, individualPosition, cs_per_min (from challenges JSONB).
  • Add GET /api/v1/summoners/{puuid}/champion-stats. Aggregate by championId: COUNT(*) as games, AVG(kills), AVG(deaths), AVG(assists), ROUND(AVG(CASE WHEN win THEN 1.0 ELSE 0.0 END) * 100, 1) as winrate. Filter to only champions with ≥ 3 games. Order by games DESC.
  • Add GET /api/v1/summoners/{puuid}/stats-overview. Return overall stats: total games, overall winrate, average KDA, most played champion ID, current win streak (count consecutive wins from most recent game backwards).
  • Cache all three new endpoints with Redis TTL 300s. Add cache invalidation for all three keys in the refresh_summoner task — same pattern as the summoner profile cache.
  • Add typed wrappers to src/lib/api.ts: getMatches(puuid, limit, offset), getChampionStats(puuid), getStatsOverview(puuid). Define TypeScript interfaces for each response shape.
# backend — champion stats query from sqlalchemy import func, case stats = await session.execute( select( MatchParticipant.championId, func.count().label('games'), func.avg(MatchParticipant.kills).label('avg_kills'), func.avg(MatchParticipant.deaths).label('avg_deaths'), func.avg(MatchParticipant.assists).label('avg_assists'), (func.avg(case((MatchParticipant.win, 1.0), else_=0.0)) * 100).label('winrate'), ) .where(MatchParticipant.puuid == puuid) .group_by(MatchParticipant.championId) .having(func.count() >= 3) .order_by(func.count().desc()) )

Champion Data — CDragon Integration
~1.5 hrsSetup
  • Your DB stores champion IDs (integers) but the UI needs names and images. CDragon is the free, no-auth alternative to the official Data Dragon. Fetch champion data from: https://cdn.merakianalytics.com/riot/lol/resources/latest/en-US/champions.json — this gives you ID → name + image mapping.
  • Create src/lib/champions.ts. Fetch and cache the champion list at build time using Next.js fetch caching: fetch(url, { next: { revalidate: 86400 } }) — revalidates daily. Export a getChampionById(id: number) helper and a getChampionImageUrl(id: number) helper.
  • Champion images come from Data Dragon CDN: https://ddragon.leagueoflegends.com/cdn/14.10.1/img/champion/{ChampionName}.png. You need the patch version — fetch it from https://ddragon.leagueoflegends.com/api/versions.json and use versions[0] (the latest). Cache this too.
  • Create src/components/ui/ChampionIcon.tsx. Props: championId: number, size?: number. Renders the champion portrait image with a gold ring border and a tooltip showing the champion name. Use Next.js <Image> component with width and height props.
  • Add ddragon.leagueoflegends.com and cdn.merakianalytics.com to next.config.ts under images.remotePatterns — Next.js blocks external images by default.
// next.config.ts — allow external champion images const nextConfig = { images: { remotePatterns: [ { protocol: 'https', hostname: 'ddragon.leagueoflegends.com' }, { protocol: 'https', hostname: 'cdn.merakianalytics.com' }, ] } }
